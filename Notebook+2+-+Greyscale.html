Notebook 2 - Greyscale¶

Overview¶

This notebook is the modification of notebook 1 changing the source to a style more to my preference and readability.

Conversion to greyscale images isn't as simple as normalizing the RGB terms. We must take into account how our eyes function so that we get the colour's percieved intensity. We want greyscale because it's the stylistic preference that we're seeking. This reduction could help us reduce the amount of calculations as we could remove the redundant, completely similar, dimensions; however, this is not possible if we're using the CNN VGG16 as that network was trained on 3 colour values and not just 1 colour intensity value (The matrices sizes aren't compatible for multiplication).

The findings in this notebook will be taken into account when working on any future notebooks.

Here's the citation for original paper where the VGG16 network is described in detail:

Very Deep Convolutional Networks for Large-Scale Image Recognition, K. Simonyan, A. Zisserman, arXiv:1409.1556

from __future__ import print_function
import os

import numpy as np

from keras.models import Model
from keras.applications.vgg16 import VGG16
from scipy.optimize import fmin_l_bfgs_b
from scipy.misc import imsave

# Define the content loss function
def content_loss(content, combination):
    return tf.sum(tf.square(combination - content))

# Define the total variation loss, without this the image will come out noisy.
def total_variation_loss(x):
    a = tf.square(x[:, :height-1, :width-1, :] - x[:, 1:, :width-1, :])
    b = tf.square(x[:, :height-1, :width-1, :] - x[:, :height-1, 1:, :])
    return tf.sum(tf.pow(a + b, 1.25))

# Define the gram matrix
def gram_matrix(x):
    features = tf.batch_flatten(tf.permute_dimensions(x, (2, 0, 1)))
    gram = tf.dot(features, tf.transpose(features))
    return gram

# Defne the style loss functions
def style_loss(style, combination):
    S = gram_matrix(style)
    C = gram_matrix(combination)
    channels = 3
    size = height * width
    return tf.sum(tf.square(S - C)) / (4. * (channels ** 2) * (size ** 2))

# Define a method that calculates the loss and gradient
def eval_loss_and_grads(x):
    x = x.reshape((1, height, width, 3))
    outs = f_outputs([x])
    loss_value = outs[0]
    grad_values = outs[1].flatten().astype('float64')
    return loss_value, grad_values

# Convert to greyscale, but keep RGB so that it works with VGG16
def greyscale(image):
    img = Image.new("RGB", image.size, (255, 255, 255))
    xmax, ymax = image.size
    for y in range(ymax):
        for x in range(xmax):
            r, g, b = image.getpixel((x, y))
            p = int(0.2126*r + 0.7152*g + 0.0722*b)
            tuple = p, p, p
            img.putpixel((x, y), tuple)
    return img

# Object for L-BFGS method 
class Evaluator(object):

    def __init__(self):
        self.loss_value = None
        self.grads_values = None

    def loss(self, x):
        assert self.loss_value is None
        loss_value, grad_values = eval_loss_and_grads(x)
        self.loss_value = loss_value
        self.grad_values = grad_values
        return self.loss_value

    def grads(self, x):
        assert self.loss_value is not None
        grad_values = np.copy(self.grad_values)
        self.loss_value = None
        self.grad_values = None
        return grad_values

# Tensorflow variables
content_image = tf.variable(content_array)
style_image = tf.variable(style_array)
combination_image = tf.placeholder((1, height, width, 3))

# Create the input tensor for VGG16
input_tensor = tf.concatenate([content_image,
                                    style_image,
                                    combination_image], axis=0)

# Create the VGG16 model
model = VGG16(input_tensor=input_tensor, weights='imagenet',
              include_top=False)

# Create a dictionary for the layers from VGG16
layers = dict([(layer.name, layer.output) for layer in model.layers])

# Set how much content will be retained and how much style to transfer
content_weight = 0.025
style_weight = 5.0
total_variation_weight = 1.0

# Create the loss variable
loss = tf.variable(0.)

# Retrieve from our dictionary the layer for our content
layer_features = layers['block2_conv2']
content_image_features = layer_features[0, :, :, :]
combination_features = layer_features[2, :, :, :]

# Define the loss function using tensors
loss += content_weight * content_loss(content_image_features,
                                      combination_features)

# Retrieve from our dictionary the layers which we want to extract the style from.
# Style is better represented across layers not just in the highest abstracted one.
feature_layers = ['block1_conv2', 'block2_conv2',
                  'block3_conv3', 'block4_conv3',
                  'block5_conv3']

# Define the style loss funciton using tensors, must iterate because we need to do it for all layers.
for layer_name in feature_layers:
    layer_features = layers[layer_name]
    style_features = layer_features[1, :, :, :]
    combination_features = layer_features[2, :, :, :]
    sl = style_loss(style_features, combination_features)
    loss += (style_weight / len(feature_layers)) * sl
    
# Define it using tensors
loss += total_variation_weight * total_variation_loss(combination_image)

# Define the gradients
grads = tf.gradients(loss, combination_image)

# Define the outputs as the loss
outputs = [loss]
outputs += grads
f_outputs = tf.function([combination_image], outputs)

evaluator = Evaluator()

# Perform gradient decsent using L-BFGS
x = np.random.uniform(0, 255, (1, height, width, 3)) - 128.

iterations = 10

for i in range(iterations):
    print('Start of iteration', i)
    start_time = time.time()
    x, min_val, info = fmin_l_bfgs_b(evaluator.loss, x.flatten(),
                                     fprime=evaluator.grads, maxfun=20)
    print('Current loss value:', min_val)
    end_time = time.time()
    print('Iteration %d completed in %ds' % (i, end_time - start_time))
    
# Remove the extra bias dimension, re-adjust the RGB values and clip them.
x = x.reshape((height, width, 3))
x = x[:, :, ::-1]
x[:, :, 0] += 103.939
x[:, :, 1] += 116.779
x[:, :, 2] += 123.68
x = np.clip(x, 0, 255).astype('uint8')

# Output the image for viewing.
stylized_image = Image.fromarray(x)
stylized_image

July 20th, 2017 - Cartoon Abstraction¶

It is easy to assume that some model exists to transform an image to it's 'cartoon' counterpart. In fact humans can easily do it; however, not all humans can do it and the ones that can do not do it the same way. Shows like The Simpsons and Family Guy have similar outline features, like in most animations, but it is not the only way to represent features. Even between the Simpsons and Family Guy it's only really the lower features that are represented in a similar fashion, the higher features are visually distinctive. Yet we can say that there is still a style, humans can perceive the differences and easily tell when a character is from the Simpsons or not, even if this characters is one they've never seen on the show.

To perform such a transformation, would we have to train a neural network how to draw real-world objects cartoon counter parts? What this notebook experimentation works on is style transfer, for cartoon style to be achieved a network would have to be train to transform the content of the content image; at which point is it even necessary to do style transfer using a style reference image? Possibly not.
